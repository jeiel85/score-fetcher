## 🚀 주요 마일스톤 (Timeline)

### 🌑 2026-03-26: 대대적인 리브랜딩 및 사용성 고도화
- **리브랜딩**: 서비스 명칭을 **"콘티 메이커 (Conti Maker)"**로 확정.
- **아이덴티티**: 블랙 테마 기반의 프리미엄 아이콘 및 소셜 공유 카드 제작 적용.
- **모바일 대응**: 관리자 페이지(`admin.html`)의 반응형 레이아웃 구현 (한 손 관리 가능).
- **데이터 무결성**: 가사 오류 신고 시스템 구축하여 사용자 제보 기반 데이터 보정 체계 마련.

### 🗄️ 2026-03-25: 데이터 아키텍처 혁신 (Firebase 마이그레이션)
- **로컬 -> 클라우드**: 정적 파일(`hymn_list.txt`, `lyrics.json`) 기반에서 Firebase Realtime DB 기반으로 데이터 전환.
- **SWR 도입**: `localStorage` 캐시와 실시간 DB를 조합한 **Stale-While-Revalidate** 패턴 적용 (0ms 로딩 실현).
- **관리자 구축**: 역대 콘티 관리, 개별 곡/가사 등록 기능을 포함한 통합 관리자 대시보드 탄생.

### 💎 2026-03-24: 디자인 리뉴얼 (Digital Maestro)
- **UI/UX**: Apple 스타일의 세련된 다크 테마 및 글래스모피즘(Glassmorphism) 적용.
- **인터렉션**: 부드러운 스크롤, 애니메이션, 퀵 인덱스 바 추가로 탐색 경험 극대화.

## 💡 주요 기술적 결정 (Key Decisions)

1.  **Vanilla JS vs Framework**: 
    - 가벼움과 빠른 프로토타이핑을 위해 React 등 프레임워크 없이 바닐라 JS로 개발 유지.
    - 대규모 상태 관리를 위해 Firebase Realtime DB를 적극 활용함.
2.  **보안 전략**:
    - 일반 사용자는 익명 로그인(Anonymous)으로 가볍게 진입.
    - 관리자는 화이트리스트 기반 Google OAuth 인증 채택.
3.  **오프라인 전략**:
    - 예배 현장의 불안정한 네트워크를 고려하여 모든 텍스트 데이터(곡/가사)는 기기 로컬에 1차 캐싱함.
