## 버전 관리 정책

| 구분 | 조건 | 예시 |
|------|------|------|
| **PATCH** | 버그 수정, 소규모 CSS 개선 | v1.6.1 → v1.6.2 |
| **MINOR** | 새 기능 추가 | v1.5.x → v1.6.0 |
| **MAJOR** | 대규모 리팩토링 / 구조 변경 | v1.x → v2.0 |

- PR 커밋 메시지에 버전 태그 포함
- MINOR 이상에서 GitHub Release 태그 생성
- 세션 간 버전 유지: `CLAUDE.md` + `MEMORY.md` 동기화로 관리
