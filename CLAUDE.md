# score-fetcher 프로젝트

## 프로젝트 개요
교회 예배 콘티(악보 세트리스트)를 관리하고, 악보 이미지를 자동으로 불러오는 단일 페이지 앱입니다.

## Firebase 구조
- Realtime Database URL: `https://score-fetcher-db-default-rtdb.firebaseio.com`
- `/history` — 콘티 이력 저장 (POST: 저장, DELETE /{key}: 삭제)
- `/fcm_tokens` — FCM 푸시 토큰 목록
- **보안 규칙**: `auth != null` — 인증된 사용자만 읽기/쓰기 허용
- **익명 인증**: 앱 로드 시 자동 `signInAnonymously()`, 모든 REST 요청에 `?auth=<idToken>` 포함

## 주요 기능

### 악보 / 콘티 관리
- 찬양 번호 입력 → 악보 이미지 자동 표시 (버튼명: **악보 만들기**)
- **제목 자동 치환**: 번호만 입력해도 `hymn_list.txt` 기반으로 제목 자동 완성 (앱 로드 시 미리 캐시)
- 악보 전체화면 뷰어 (좌우 스와이프, 하단 스와이프 닫기, 풀블리드, pull-to-refresh 차단)
- **전체화면 UI 오버레이 자동 숨김**: 헤더(제목)·푸터(좌우 버튼)가 악보 위에 오버레이로 표시 → 3초 후 자동 페이드아웃
  - 악보 넘길 때마다 UI 재표시 후 3초 타이머 리셋
  - 화면 탭으로 UI 토글 (숨김 ↔ 표시), 더블탭·스와이프·버튼 탭은 토글 제외
- **핀치 줌**: 전체화면 뷰어 + 가로 분할 뷰어 모두 지원 (1~4x, 확대 중 패닝, 더블탭 리셋)
- **세로 모드 카드 3열 그리드**: 전체 악보 순서 한눈에 파악 (480px 이하 2열), `#result-container`는 `<main>` 하단에 위치
- **드래그 앤 드롭**: ⠿ 핸들 터치로 카드 순서 변경 → textarea·sheetList 자동 동기화
- **가로 모드**: 악보 만들기 시 `activateLsView()` 자동 호출 → ls-active 분할 뷰 즉시 전환 (카드 그리드 미표시)
  - `activateLsView()`: ls-active 클래스 추가 + 콘티 이미지 생성 + 현재 보던 악보(`currentSheetIndex`) 이어서 표시, 없으면 첫 번째 (`js/sheet-viewer.js`)
- **세로 전체화면 → 가로 회전**: 전체화면 닫고 ls-active 분할뷰 자동 전환 (보던 악보 이어서 표시)
- 가로→세로 회전 시 ls-active 해제 + 카드 그리드 재생성 (전체화면 자동 오픈 없음)
- 세로→가로 회전 시 카드가 있으면 ls-active 자동 활성화 (`_lsUserDismissed` 아닐 때)
- 가로 모드에서 사용자가 직접 닫으면 resize 이벤트로 재활성화 차단 (`_lsUserDismissed` 플래그)
- 악보 만들기 완료 후 저장·공유 버튼 영역으로 자동 스크롤 (세로 모드만)
- 이력 불러오기 시 악보 초기화 + 첫 카드 자동 스크롤
- **초기화** 버튼은 상단 버튼 그룹에 위치

### 콘티함 (이력)
- Firebase `/history`에 저장 (버튼명: **콘티함에 저장**)
- 이력 모달 (버튼명: **콘티함**): 불러오기(탭) / 삭제(길게 누르기, 700ms)
- 이력 항목 길게 누르기 시 네이티브 팝업 차단 (`user-select:none`, `contextmenu` 차단)
- 이력 검색 기능 (`renderHistoryList` 기반, 전체 캐시 후 필터)
- 이력 항목 레이아웃: `display:flex; text-align:left` 명시, 내용 좌정렬 통일
- 콘티 순번 정확화 / 이중개행 제거

### 찬양 목록 (`hymn_list.txt` + `lyrics.json`)
- 전체 곡 목록 모달 (버튼명: **찬양 목록/추가**)
- **통합 검색**: 번호·제목(`hymn_list.txt`) + 가사·태그(`lyrics.json`) 동시 검색
- 클릭으로 입력창에 추가
- **가사 버튼**: 가사 데이터가 실제로 있는 곡에만 표시 (없으면 버튼 자체 숨김)
  - 클릭 시 가사 모달: 상단 태그 칩(파란 배지), 하단 가사 전문
- **악보 버튼** (`.btn-score`): 이미지 파일이 존재하는 곡에만 표시 (`images/{numPadded}.jpg` 확인)
  - 클릭 시 `openScorePreview()` — 기존 fullscreenViewer 재활용, 네비 버튼 숨김
  - `js/sheet-viewer.js`의 `openScorePreview(numPadded, displayTitle)` 함수 사용

### 가사 데이터 형식 (`lyrics.json`)
- **크롤링 현황**: 690곡 중 553곡 가사 수집 완료 (137곡 실패 — 사이트 미등록 곡)
```json
{
  "metadata": { "total": 690, "crawled": 553, "failed": 137, "last_updated": "..." },
  "songs": [
    {
      "id": 1,
      "number": "001",
      "title": "...",
      "subtitle": "...",
      "qs_idx": 101,
      "lyrics": "1절 가사\n\n후렴 가사",
      "tags": ["태그1", "태그2"],
      "status": "done",
      "crawled_at": "..."
    }
  ]
}
```
- `number`: 3자리 0패딩 문자열 (`"001"`)
- `lyrics: null` 항목은 가사 모달에서 자동 제외
- 구형 단순 맵 형식 `{"001": "가사..."}` 하위 호환 유지
- 실 데이터 파일로 `lyrics.json`을 덮어쓰면 즉시 반영 (코드 수정 불필요)

### 공유
- 콘티 이미지 생성 및 공유 (`shareConti`) — 버튼명: **콘티 공유하기**
- 디자인: 어두운 단색 풀블리드 배경 (제목 해시 기반 8가지 팔레트 중 랜덤 선택, 콘티마다 일관)
- 날짜(`YYYY-MM-DD`) 부분 액센트 컬러 + 언더라인, 곡 번호 액센트 컬러
- 헤더(`| ... |`) 반투명 흰색, 코멘트(`-`) 살짝 작고 반투명
- Web Share API 지원, 폴백 시 이미지 다운로드
- 미리보기 모달: 헤더 없음, 이미지 풀스크린 + 하단 푸터 오버레이
  - 푸터 구성: `[닫기 ✕]` · `[공유하기 🚀]` (좌: 닫기, 우: 공유)
  - 모달 열리면 3초 후 푸터 자동 숨김 (스르륵)
  - 이미지 터치 시 토글: 숨김↔표시 (표시 후 3초 자동 숨김)
  - **딥링크 칩**: 미리보기 오픈 시 Firebase 저장 → 딥링크 URL 칩 표시 (탭하면 클립보드 복사)
- **딥링크 공유 플랫폼별 동작**:
  - iOS/Safari: `{ title, url, files }` → 카카오톡에 이미지 + 링크 함께 공유
  - Android/Chrome: `files+url` 동시 미지원 → 이미지만 공유 + 클립보드 자동 복사 + 토스트 안내
  - ⚠️ `text`에 URL 포함 금지: Android OS가 URL 추출 → 카카오톡에 중복 표시됨
  - ⚠️ `title`+`text` 중복 금지: iOS KakaoTalk에서 빈 줄 삽입됨

### 알림 / 설치
- FCM 푸시 알림 (Firebase SDK, VAPID 설정 완료)
- 새 콘티 등록 시 알림:
  - 제목: `새 콘티가 등록되었습니다 ♬` (고정)
  - 본문: `[콘티제목] 곡1 · 곡2 · ...` (곡 줄 `^\d+` 최대 4개)
  - **발신자 본인 제외**: 저장 시 `senderToken` 포함 → Cloud Function에서 필터
  - **중복 방지 (Cloud Function)**: `/fcm_tokens` 내 동일 토큰 dedup 후 발송
  - **중복 방지 (클라이언트)**: 토큰 등록 시 동일 토큰 값 가진 다른 키(구형 `dev_` 등) 자동 DELETE
  - **device_id**: 기기 핑거프린트 기반 `fp_` 키 → 같은 기기는 항상 동일 키로 PUT (덮어쓰기)
  - **tag 고유화**: `new-conti-${Date.now()}` — Samsung 조용히 교체 버그 방지
- Cloud Function (`functions/index.js`): `/history/{id}` 생성 감지 → 중복 제거된 토큰 발송 → 만료 토큰 정리
  - 리전: `us-central1` (asia-northeast3는 Realtime DB 트리거 미지원)
- **포그라운드 알림**: 앱 열려있을 때도 `registration.showNotification()`으로 시스템 알림 표시
- **알림 클릭 시 자동 로드**: 최신 콘티 불러오기 + 악보 만들기 자동 실행
  - 앱 닫힌 경우: `/?autoload=1` URL 파라미터 감지
  - 앱 열린 경우: SW → 페이지 `postMessage({ type: 'AUTOLOCAD_LATEST' })`
- **서비스워커 즉시 활성화**: `skipWaiting()` + `clients.claim()`
- 푸시 알림 아이콘: `icon.png` (컬러 대형), `badge.svg` (흰색 음표, 모노크롬, 상태바용)
- **앱 설치 버튼**: standalone 모드가 아니면 항상 표시, 브라우저별 안내 분기
  - 삼성 인터넷 → "Chrome으로 열기" 안내
  - 카카오톡 인앱 → "기본 브라우저로 열기" 안내
  - iOS Safari → "홈 화면에 추가" 안내
  - Firefox → "Chrome/Edge 사용" 안내
  - Chrome(지원) → 자동 설치 프롬프트
- 알림 활성화 버튼 (🔔)

### UX
- **화면 꺼짐 방지** (Wake Lock API) — 항상 자동 ON, UI 버튼 없음
- 카카오톡 인앱 브라우저 감지 → 외부 브라우저 유도 배너
- Open Graph 메타태그 (링크 미리보기)
- 안드로이드 뒤로 가기: 모달 닫기 → 두 번 누르면 종료 토스트
- 모달 외부 영역 터치 시 닫기 (찬양목록·콘티함·가사 모달; 공유 미리보기 제외)
- 가로 화면 대응: 모달 `max-width: 540px`, 공유 미리보기 이미지 `object-fit: contain`
- 모바일 터치 우선 UX

## 🔖 버전 관리 규칙 (반드시 준수)

> 버전이 파일마다 달라지는 혼란을 방지하기 위한 규칙. 코드 변경 시 아래 4곳을 **동시에** 업데이트할 것.

### 버전 번호 체계 (Semantic Versioning)
```
v[MAJOR].[MINOR].[PATCH]
```
| 구분 | 조건 | 예시 |
|------|------|------|
| PATCH | 버그 수정, 텍스트/스타일 소폭 수정 | v1.7.1 → v1.7.2 |
| MINOR | 새 기능 추가, UX 구조 변경 | v1.7.x → v1.8.0 |
| MAJOR | 전체 리팩토링, 구조적 대개편 | v1.x.x → v2.0.0 |

### 버전 기재 위치 (5곳 동시 수정)

> ⚠️ `js/utils.js`의 `APP_VERSION`이 런타임에 모든 `.app-version-badge`를 덮어씀.
> `index.html` 배지 HTML 텍스트는 초기 렌더에만 잠깐 보이며, 실제 화면 표시 버전은 `utils.js`가 기준.

| 파일 | 위치 | 형식 |
|------|------|------|
| `js/utils.js` | `APP_VERSION` / `BUILD_DATE` 상수 | `"v1.8.1"` / `"2026.04.08"` |
| `index.html` | JS `const VERSION = '...'` | `'1.8.1'` (숫자만, 강제 새로고침 판별용) |
| `index.html` | `<span class="app-version-badge">` (2곳) | `v1.8.1` (HTML 초기값) |
| `README.md` | 상단 배지 `version-vX.X.X` | `v1.8.1` |
| `README.md` | 상단 배지 `build-YYYY.MM.DD` | `2026.04.08` |

### 버전 업 시 체크리스트
```

[ ] index.html  — const VERSION = '새버전'                      ← 강제 새로고침 판별
[ ] index.html  — app-version-badge 텍스트 (2곳, replace_all)   ← HTML 초기값
[ ] README.md   — 배지 version-v새버전, build-날짜
[ ] README.md   — 🔄 변경이력 테이블 맨 위에 새 행 추가 (bold 강조)
[ ] CLAUDE.md   — 현재 상태 테이블 날짜·버전·최신 커밋 업데이트
[ ] CLAUDE.md   — 변경 이력 섹션 맨 위에 새 항목 추가
[ ] git commit  — "fix/feat: #이슈번호 설명" 형식
[ ] git push    — origin main
```

### 규칙 위반 패턴 (금지)
- ❌ `js/utils.js` 미수정 — 화면에 구버전이 표시됨 (가장 흔한 누락)
- ❌ 한 파일만 버전 업 후 커밋 (나머지 파일과 불일치 발생)
- ❌ `VERSION` 상수는 올리고 배지는 그대로 두거나 그 반대
- ❌ README 변경이력에 새 버전 추가 없이 배지만 변경
- ❌ CLAUDE.md 현재 상태 테이블을 업데이트하지 않음

| **`currentSheetIndex` 초기값** | `-1` (한 번도 악보를 직접 선택하지 않은 상태). `0`으로 바꾸면 "아직 선택 안 함" 판별 불가. |
| **`_scorePreviewFromModal` 플래그** | 찬양목록에서 악보 미리보기 열었을 때 `true`. `closeFullscreen()` 시 `openSongModal()` 재호출. |
