# 🤖 에이전트 개발 지침 (Project Agents Guide)

이 문서는 **콘티 메이커 (Score Fetcher)** 프로젝트를 작업하는 AI 에이전트들이 반드시 숙지해야 할 핵심 지침과 과거의 시행착오를 정리한 것입니다. 다음 세션의 에이전트는 이 내용을 바탕으로 프로젝트의 일관성을 유지해야 합니다.

---

## 🚀 1. 프로젝트 핵심 철학 및 아키텍처
- **No Framework**: React/Vue 없이 **Vanilla JS**만 사용합니다. 복잡한 상태 관리는 DOM 상태와 `localStorage`, Firebase Realtime DB를 조합하여 해결합니다.
- **REST over SDK**: Firebase Database는 용량 절감을 위해 SDK 대신 직접적인 **REST API(fetch)** 호출을 기본으로 합니다. (단, FCM 알림 기능은 SDK 사용)
- **Offline First**: 모든 텍스트 데이터(곡 목록, 가사)는 기기 로컬에 1차 캐싱(`localStorage`)하여 네트워크 불안정 시에도 0ms 로딩을 보장합니다.

## ⚠️ 2. 절대 불변의 UX 원칙 (UX Immutables)
*이 규칙을 위반하면 앱의 사용성이 크게 훼손되며, 과거에 반복적으로 발생했던 버그들입니다.*

1.  **전체화면 뷰어 자동 오픈 금지**: 세로 모드에서 전체화면 뷰어(`fullscreenViewer`)는 사용자가 악보 카드를 **직접 클릭**할 때만 열려야 합니다. 화면 회전이나 검색 완료 이벤트로 자동 오픈되는 것을 엄격히 금지합니다.
2.  **가로 모드(Landscape) 자동 전환**: 세로 모드에서 악보를 만든 후 기기를 가로로 회전하면, 전체화면 뷰어는 닫히고 **가로 분할 뷰(`ls-active`)**가 자동으로 활성화되어야 합니다. 이때 보던 악보 번호(`currentSheetIndex`)가 유지되어야 합니다.
3.  **`currentSheetIndex` 초기값은 `-1`**: `0`으로 설정하면 첫 번째 곡을 "선택한 상태"와 "아직 아무것도 선택하지 않은 상태"를 구분할 수 없어 자동 로직이 꼬입니다.
4.  **UI 오버레이 자동 숨김**: 악보 뷰어(전체화면/가로 분할)의 헤더와 버튼들은 3초 후 자동으로 페이드아웃되어야 하며, 화면 탭으로 토글 가능해야 합니다.

## 🛠️ 3. 반복된 버그 및 해결책 (Lessons Learned)

| 현상 | 원인 | 해결책 |
| :--- | :--- | :--- |
| **관리자 로그아웃 현상** | 메인 앱의 익명 로그인 호출이 관리자 세션을 덮어씀. | 관리자 페이지에서는 익명 로그인 호출을 차단하고 화이트리스트 기반 Google OAuth만 사용. |
| **데이터 유실 (DB 수정 시)** | `PUT` 요청으로 데이터 전체를 덮어씌움. | `PATCH` 방식을 사용하여 기존 데이터를 보존하고 필요한 필드만 업데이트. |
| **스크롤 먹통** | 인덱스 바 추가 시 부모 컨테이너의 `overflow: hidden` 설정 오류. | 스크롤 영역과 레이아웃 영역의 CSS 상속 구조를 명확히 분리. |
| **푸시 알림 중복** | 동일 기기에서 여러 브라우저 사용 시 토큰 중복 생성. | 기기 핑거프린트(`device_id`) 기반으로 토큰 키를 고정하여 항상 덮어쓰기(`PUT`) 처리. |
| **카톡 공유 링크 중복** | `text` 파라미터에 URL을 포함하면 Android 카톡에서 중복 표시됨. | `text`에서 URL을 제거하고 `url` 파라미터만 사용. |

## 🔢 4. 버전 동기화 프로토콜 (5-Point Sync)
코드 수정 후 배포 전, 다음 5곳의 버전을 **반드시 동시에** 업데이트해야 합니다.

1.  `js/utils.js`: `APP_VERSION`, `BUILD_DATE` (실제 화면 표시용)
2.  `index.html`: `const VERSION` (캐시 무효화 및 강제 새로고침 판별용)
3.  `index.html`: `.app-version-badge` 텍스트 (초기 HTML 렌더용)
4.  `README.md`: 상단 배지 및 변경이력 테이블
5.  `CLAUDE.md`: 현재 상태 테이블 및 변경 이력

## 📋 5. 배포 및 작업 필수 절차
1.  **작업 전 `git pull origin main`**: 항상 최신 소스에서 시작합니다.
2.  **한글 깨짐 주의**: `pr-body.md` 등 문서 파일 수정 시 UTF-8 인코딩을 유지합니다.
3.  **문서 동기화**: `source_history.md`를 먼저 갱신하여 작업 방향을 명확히 한 후 코드를 수정합니다.
4.  **관리자 기능 테스트**: `admin.html` 수정 시 Google 로그인 및 Firebase 통신 여부를 반드시 확인합니다.
